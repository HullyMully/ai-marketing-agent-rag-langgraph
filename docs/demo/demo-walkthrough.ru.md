# Разбор демо

[🇺🇸English](./demo-walkthrough.md) | 🇷🇺Русский

Сквозные сценарии для **настраиваемой платформы AI-ассистента поддержки**,
сконфигурированной здесь с поставляемым примером профиля (Acme Growth Studio). Всё
работает в офлайн-режиме (`MOCK_LLM=true`, мок-эмбеддинги), ключи API не нужны. Все
данные демо вымышлены и используют домены `.example`.

В каждом сценарии ответ проходит через один и тот же конвейер: **LLM-планировщик →
валидация на бэкенде → финальный ответ**. Детерминированный офлайн-движок следует
тем же правилам, поэтому разбор воспроизводим без модели.

## Настройка

```bash
pip install -r requirements.txt
cp .env.example .env
python scripts/ingest_knowledge.py        # индексировать базу знаний
uvicorn app.main:app --reload             # API на http://localhost:8000
# (опционально) python scripts/seed_demo_data.py
```

Откройте веб-интерфейс `http://localhost:8000/demo`, Swagger `/docs` или
используйте `curl` ниже. Сохраняйте **один `session_id`** внутри сценария, чтобы
задействовать память диалога.

## Сценарий 1 — Вопросы по знаниям (RAG)

**Пользователь:** «Какие услуги вы предоставляете?», затем «Какие есть тарифные
пакеты?»

**Ожидается:** планировщик отвечает из базы знаний (RAG по `services.md` /
`pricing.md`); `knowledge_used` = true, в `sources` — файлы. Пользователь видит
естественный краткий ответ, а не сырые чанки. Ни лида, ни тикета.

```bash
curl -X POST localhost:8000/chat -H "Content-Type: application/json" \
  -d '{"session_id":"s1","user_message":"Какие услуги вы предоставляете?"}'
# > knowledge_used: true · sources: ["services.md", ...]
```

## Сценарий 2 — Светская/мета/эмоциональная реплика ничего не создаёт

**Пользователь (любое из):** «Hello» · «SAY HELLO TO ME» · «Another one» ·
«What do you mean?» · «EXPLAIN WHAT DO YOU MEAN» · брань.

**Ожидается:** каждый раз естественный ответ, **ни лида, ни тикета, без
повторяющегося меню, без скриптовых заглушек**. Если пользователь продолжает
путаться или раздражён, квалификация ставится на паузу, а не навязывает форму.

---

## Сценарий 3 — Растерянный пользователь, память сохраняется

**Пользователь (одна сессия):**
1. «Hello I need help with marketing»
2. «I don't know. Help me with that»
3. «Help starting a project please»
4. «I want paid ads for my SaaS»
5. «I don't remember»
6. «Oh sorry, company is FalkoTeam»
7. «FalkoTeam, I told you»

**Ожидается:** ассистент помнит **FalkoTeam** и интерес к paid ads/SaaS, обновляет
черновик лида, **не** перезапускает поток и не повторяет один и тот же вопрос,
**не** создаёт лид (нет имени/email/бюджета) и тикет.

---

## Сценарий 4 — Пользователь становится лидом

**Пользователь (одна сессия):**
1. «I want to start a project»
2. «Paid ads for my SaaS»
3. «Company is BrightDesk, budget around $5k/month»
4. «My name is Sam, email sam@brightdesk.example»

**Ожидается:** до последнего сообщения лида нет (поля неполные). После сообщения 4
бэкенд валидирует и создаёт **ровно один** лид с BrightDesk, Sam,
`sam@brightdesk.example`, paid ads, SaaS и ~$5k/month. Ассистент подтверждает
создание только **после** подтверждения бэкендом; повтор деталей не создаёт
дубликат.

```bash
curl localhost:8000/crm/leads   # один лид для этой сессии
```

**Скриншот:** `docs/screenshots/web-chat-lead-flow.png`

---

## Сценарий 5 — Эскалация на человека

**Пользователь:** «I need a human manager»

**Ожидается:** бэкенд создаёт высокоприоритетный **тикет эскалации**, и ассистент
подтверждает передачу — только **после** создания тикета. В правой панели —
реальный тикет; до этого никакой фейковой карточки тикета нет.

```bash
curl localhost:8000/tickets
```

## Что демонстрирует каждый сценарий

| Сценарий | Демонстрирует |
|----------|---------------|
| 1 | RAG-поиск + обоснованные ответы с источниками |
| 2 | Нет лида/тикета на светский/мета/грубый ввод; без скриптовых заглушек |
| 3 | Память сессии + черновик лида; без повторов вопросов |
| 4 | Валидированное создание единственного лида при всех полях |
| 5 | Эскалация, подтверждённая бэкендом (без фейковых карточек) |
| 6 | Метрики/наблюдаемость по сохранённым данным |
