# Веб-демо

[🇺🇸English](./web-demo.md) | 🇷🇺Русский

Приложение FastAPI отдаёт небольшой тёмный веб-интерфейс рядом с API. Все страницы
используют единую дизайн-систему, поэтому лендинг, чат, обзор API и метрики
выглядят как один внутренний продукт.

## Запуск

```bash
pip install -r requirements.txt
cp .env.example .env                  # по умолчанию офлайн-режим (без API-ключей)
python scripts/ingest_knowledge.py    # индексация базы знаний
uvicorn app.main:app --reload
```

## Страницы

- `http://localhost:8000/` — лендинг
- `http://localhost:8000/demo` — чат-демо
- `http://localhost:8000/api-overview` — оформленный обзор API (продуктовый вид эндпоинтов)
- `http://localhost:8000/metrics` — дашборд метрик (читает `/metrics/demo`)
- `http://localhost:8000/docs` — Swagger (без изменений)

## Чат-демо

Каждый ответ приходит от реального агента (`POST /chat`). Кнопки быстрых действий
просто отправляют обычное сообщение — готовых сценариев нет, дальше вы продолжаете
диалог сами. Каждый ход решает **LLM-планировщик**: он читает профиль компании,
релевантные знания, недавнюю историю, память сессии, черновик лида и состояние
тикета и возвращает структурированное решение. **Бэкенд валидирует** это решение
перед любым действием, поэтому ассистент естественно квалифицирует лиды,
обрабатывает приветствия, шутки, замешательство и раздражение и не навязывает
скрипт.

- **Services Q&A** / **Pricing from RAG** — ответ из базы знаний
- **Lead creation flow** — планировщик собирает данные по ходу диалога; бэкенд
  создаёт лид в CRM только когда известны имя, компания, email, услуга и бюджет
- **Human escalation** — создаёт тикет для менеджера, когда эскалация обоснована
- **Memory follow-up** — ассистент вспоминает данные, собранные ранее

Правая панель **Conversation state** показывает живой вывод планировщика: текущий
**режим** (answering / exploring / qualifying / paused), распознанный **интент**,
**известные интересы**, **черновик лида** с известными и недостающими полями,
**последнее действие**, **следующий шаг** и использованные **источники знаний**.
Лид помечается как *создан* только когда его действительно создал бэкенд — в
ознакомительных диалогах фейковые данные лида не показываются.

## Реальная LLM

По умолчанию демо работает через реальную LLM (`MOCK_LLM=false`). Настройте
OpenAI-совместимый эндпоинт — например DeepSeek:

```bash
MOCK_LLM=false
OPENAI_BASE_URL=https://api.deepseek.com
OPENAI_API_KEY=...      # только в окружении; не логируется и не коммитится
LLM_MODEL=deepseek-chat
```
