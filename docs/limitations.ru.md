# Ограничения

[🇺🇸English](./limitations.md) | 🇷🇺Русский

Проект — это **MVP / прототип для портфолио**, намеренно ограниченный ради
понятности и запускаемости, а не продакшен-уровня. Известные ограничения:

## ИИ / RAG
- **Мок-LLM** (`MOCK_LLM=true`) — детерминированная заглушка по шаблонам, она не
  «рассуждает». Для настоящей генерации используйте реальную OpenAI-совместимую модель.
- **Мок-эмбеддинги** — хеширующий «мешок слов» с фильтрацией стоп-слов. Стабильны и
  без зависимостей, но **семантически не сильны**. Реальные эмбеддинги дают
  заметно лучший поиск.
- **Классификация интентов** в демо-режиме основана на ключевых словах/правилах.
  Она предсказуема и удобна для тестов, но хрупка на перефразировках; путь через
  LLM устойчивее.
- **Извлечение полей** (имя/контакт и т. д.) использует простые регулярные
  выражения и пропускает необычные форматы. Извлечение через LLM со
  структурированным выводом было бы надёжнее.

## Инженерия
- **Нет аутентификации / ограничения частоты** на API.
- **SQLite** — однофайловая БД; годится для демо, но не для конкурентной нагрузки.
- **Слоты памяти сессии** хранятся в процессе и сбрасываются при перезапуске.
  История сообщений сохраняется в SQLite.
- **Мок-CRM**: лиды — это локальная таблица, моделирующая паттерн интеграции, а не
  реальная CRM (HubSpot/Pipedrive и т. п.).
- **Нет потоковой передачи** ответов; агент возвращает одно сообщение.
- **Минимальная наблюдаемость** (только базовое логирование).

## Данные и безопасность
- Всё содержимое базы знаний, лиды, тикеты и цены — **синтетические**.
- В репозитории нет секретов; токены/ключи живут только в `.env`.

Это осознанные компромиссы, чтобы проект оставался компактным, безопасным для
публикации и запускался без единого API-ключа.
